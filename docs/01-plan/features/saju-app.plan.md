# Plan: 사주팔자 웹 애플리케이션

> Status: Draft
> Created: 2026-03-30
> Feature: saju-app

---

## 1. Overview

### 1.1 목표

전통 사주명리학 기반의 모바일 반응형 웹 애플리케이션을 개발한다. 사용자의 생년월일시를 입력받아 사주팔자를 자동 계산하고, AI를 활용한 맞춤형 운세 해석을 제공한다.

### 1.2 참고 서비스

- **사주아이(saju-kid.com)**: 990원 사주, 궁합, 대운, 택일, 연도별 운세, 오늘의 무료운세, AI 채팅 상담
- **프로젝트 가이드 문서**: `사주풀이 애플리케이션 개발 완전 가이드.md`

### 1.3 타겟 사용자

- MZ세대 중심의 운세에 관심 있는 일반 사용자
- 모바일 웹 중심 (반응형 디자인)

## 2. Core Features (MVP)

### 2.1 사주팔자 자동 계산

| 항목 | 상세 |
|------|------|
| 입력 | 생년월일, 태어난 시간(선택), 성별, 양력/음력(윤달 여부) |
| 계산 | 연주, 월주, 일주, 시주 (4기둥 8글자) |
| 보정 | 한국 표준시 -30분 보정, 입춘 기준 연주 판단 |
| 출력 | 사주 명반 (천간/지지, 오행, 십성 표시) |

### 2.2 오행 분석

- 목/화/토/금/수 분포 시각화 (도넛 차트 또는 바 차트)
- 오행 균형/불균형 해석
- 신강/신약 판단

### 2.3 오늘의 운세 (무료)

- 매일 갱신되는 일일 운세
- 사주 기반 개인 맞춤형 해석
- 로그인 없이 이용 가능 (프로필 저장 시 로그인 필요)

### 2.4 AI 사주 풀이

- OpenAI GPT API 활용 맞춤형 텍스트 해석
- 주제별 풀이: 종합운, 애정운, 금전운, 직업운, 건강운
- 구조화된 프롬프트로 일관된 품질 보장

### 2.5 궁합 분석

- 두 사람의 사주 비교 분석
- 궁합 점수 및 상세 해석
- 연애/결혼/사업 궁합 구분

### 2.6 대운/세운 분석

- 10년 단위 대운 흐름 표시
- 연도별 세운 분석
- 타임라인 시각화

### 2.7 프로필 관리

- 본인/가족/지인 사주 프로필 저장
- 소셜 로그인 (Google, 카카오)
- 저장된 결과 보관함

## 3. Tech Stack

### 3.1 Frontend

| 기술 | 용도 |
|------|------|
| **Next.js 15 (App Router)** | SSR/SSG, SEO 최적화, 라우팅 |

| 기술 | 용도 |

## 4. UI/UX Design Direction

### 4.1 레이아웃 (모바일 퍼스트)

```
┌─────────────────────────┐
│  헤더 (로고 / 로그인)     │
├─────────────────────────┤
│                         │
│     메인 콘텐츠 영역       │
│                         │
├─────────────────────────┤
│ 홈 | 사주 | 운세 | 상담 | MY │  ← 하단 네비게이션
└─────────────────────────┘
```

### 4.2 주요 화면

| 화면 | 설명 |
|------|------|
| **홈** | 배너 슬라이더, 서비스 카드 목록, 오늘의 운세 미리보기 |
| **사주 입력** | 생년월일시/성별/양음력 입력 폼 |
| **사주 결과** | 사주 명반, 오행 차트, 십성, AI 풀이 |
| **궁합** | 두 사람 정보 입력 + 궁합 결과 |
| **대운/세운** | 타임라인 형태의 운세 흐름 |
| **오늘의 운세** | 일일 무료 운세 |
| **AI 상담** | 채팅 형태의 AI 사주 상담 |
| **MY** | 프로필 관리, 저장된 사주, 설정 |

### 4.3 디자인 컨셉

- **색상**: 따뜻한 골드/크림 톤 + 다크 포인트 (사주아이 참고)
- **분위기**: 전통적이면서도 현대적, 친근한 캐릭터 활용 가능
- **반응형**: 모바일 320px ~ 데스크톱 1440px 대응
- **폰트**: Pretendard (한글), 전통 느낌의 서브 폰트

## 5. Data Model

### 5.1 핵심 테이블

```
users (Supabase Auth 연동)
├── profiles (사주 프로필 - 본인/가족/지인)
│   ├── name, birth_year, birth_month, birth_day
│   ├── birth_hour, birth_minute, gender
│   ├── calendar_type (solar/lunar), is_leap_month
│   └── saju_results (계산 결과 캐시)
│       ├── 4기둥 8글자 (year_gan, year_ji, ...)
│       ├── 오행 분포, 십성 배열
│       └── 대운 데이터 (JSONB)
├── fortune_readings (AI 풀이 결과 저장)
│   ├── profile_id, reading_type, content
│   └── created_at
└── compatibility_results (궁합 결과)
    ├── profile1_id, profile2_id
    ├── score, interpretation
    └── created_at
```

### 5.2 캐싱 전략

- 사주 계산 결과는 프로필별 1회 계산 후 DB 캐시
- 오늘의 운세는 일 1회 생성 후 전체 캐시
- AI 풀이 결과는 요청별 저장

## 7. Key Considerations

### 7.1 정확도 엣지 케이스

| 케이스 | 처리 방법 |
|--------|---------|
| 입춘 전/후 연주 판단 | 절입시각 데이터 활용 |
| 한국 표준시 30분 보정 | 입력 시간 -30분 |
| 야자시(23:30~00:30) | 날짜 귀속 선택 UI |
| 음력 입력 | 양력 변환 + 윤달 처리 |
| 시간 미상 | 시주 제외 6자 분석 |

### 7.2 법적 고려

- 면책 조항: "사주는 참고용이며 결정의 절대 기준이 아닙니다"
- 개인정보처리방침: 생년월일시 수집/이용 동의
- OpenAI API 사용 관련 약관 준수

### 7.3 수익화 (향후)

- 기본 사주/오행/오늘의 운세: 무료
- AI 심층 풀이 / 궁합 / 대운 해석: 유료 (990원~)
- MVP에서는 전체 무료, 추후 과금 모델 도입

## 8. Risks & Mitigations

| 리스크 | 완화 방안 |
|--------|---------|
| manseryeok 라이브러리 정확도 | 알려진 사주와 교차 검증, 필요시 자체 계산 로직 보완 |
| AI 풀이 품질 편차 | 구조화된 프롬프트 + 결과 검증 로직 |
| 모바일 성능 | SSR/SSG 활용, 이미지 최적화, 번들 최소화 |
| 사주 계산 복잡성 | 점진적 구현 (기본 → 고급 기능 순서) |
